# PRD — 이달의 기록 (Monthly Archive)

## 1. 개요

### 제품 비전
매달 10개의 고정 카테고리에 사진, 텍스트, 링크, 영상 등을 기록하고 누적하는 개인 아카이브 웹사이트. 노션 다이어리처럼 깔끔하고 여백 있는 UI로, 모바일에서 편하게 기록하고 돌아볼 수 있다.

### 핵심 목표
- 매달 정해진 카테고리를 채우는 부담 없는 기록 루틴 만들기
- 지난 달의 기록을 쉽게 돌아볼 수 있는 개인 아카이브
- 공개되지 않는 나만의 공간

---

## 2. 사용자

- **단일 사용자** (개인 기록용)
- 로그인 없음 — UUID 기반 비공개 URL로 접근
- 모바일 우선, 데스크탑도 지원

---

## 3. 접근 방식

### URL 구조
- 앱 진입: `https://[도메인]/[UUID]`
  - UUID는 최초 배포 시 환경 변수로 고정 설정
  - 추측 불가능한 형태 (예: `/a3f8c2d1-7e4b-4f1a-9c3e-2b5d8e1f0a6c`)
  - 별도 로그인 없이 이 URL 접근 시 바로 앱 사용 가능
- URL을 모르는 사람은 접근 불가 (서버에서 UUID 불일치 시 404 반환)

---

## 4. 카테고리 명세

총 10개 고정 카테고리. 각 달의 기록은 이 카테고리들로 구성된다.

| # | 카테고리 | 설명 | 지원 미디어 |
|---|---|---|---|
| 1 | 사진 | 이달을 대표하는 사진 한 장 | 이미지 업로드 |
| 2 | 노래 | 이달에 가장 많이 들은 노래 | 링크 (유튜브 / 스포티파이 등) + 텍스트 |
| 3 | 목표 | 이달에 세웠거나 이룬 목표 | 텍스트 |
| 4 | 문장 | 이달에 마음에 남은 문장이나 글귀 | 텍스트 |
| 5 | 맛집 | 이달에 가장 인상 깊었던 식당/카페 | 텍스트 + 이미지 (선택) + 링크 (선택) |
| 6 | 장소 | 이달에 다녀온 기억에 남는 장소 | 텍스트 + 이미지 (선택) |
| 7 | 소비 | 이달에 가장 잘했다고 생각하는 소비 | 텍스트 + 이미지 (선택) + 링크 (선택) |
| 8 | 영화/드라마 | 이달에 인상 깊었던 영상 콘텐츠 | 텍스트 + 이미지 (선택) + 링크 (선택) |
| 9 | 발견 | 이달에 새로 알게 된 것 (앱, 제품, 습관 등) | 텍스트 + 링크 (선택) |
| 10 | 감사한 것 | 이달에 감사했던 순간이나 사람 | 텍스트 |

### 공통 필드 (모든 카테고리)
- **메모** (선택): 자유 형식의 부가 설명, 최대 500자

## 5. 화면 구성

### 5.1 홈 화면 (`/[UUID]`)
- 현재 달의 기록 카드 그리드 (2열) 표시
- 각 카드: 카테고리 아이콘 + 이름 + 작성 여부 표시
- 작성된 카드: 미리보기 (썸네일 or 텍스트 첫 줄) 표시
- 미작성 카드: 희미한 빈 상태 표시
- 상단: 현재 년월 표시 (예: 2026년 4월)
- 하단 네비게이션: 홈(현재 달) / 아카이브(지난 달 목록)

### 5.2 카테고리 상세 / 편집 화면
- 카드 탭 시 해당 카테고리의 상세 화면으로 이동
- 미디어 타입에 맞는 입력 UI 제공
  - 이미지: 업로드 버튼 + 미리보기
  - 링크: URL 입력 → OG 이미지/제목 자동 프리뷰
  - 텍스트: 멀티라인 입력창
- 메모 입력창 (공통)
- 저장 / 삭제 버튼
- 이미 작성된 경우 기존 내용 불러와서 수정 가능

### 5.3 아카이브 화면 (`/[UUID]/archive`)
- 지난 달 기록 목록 (최신순 리스트)
- 각 항목: 년월 + 작성된 카테고리 수 표시 (예: `2026년 3월 — 8/10`)
- 탭하면 해당 달의 홈 화면으로 이동 (읽기 전용)

### 5.4 지난 달 보기 (`/[UUID]/[YYYY-MM]`)
- 특정 달의 기록을 홈 화면과 동일한 레이아웃으로 표시
- **읽기 전용** — 수정 불가
- 상단에 해당 년월 표시

---

## 6. UX 원칙

- **모바일 우선**: 기본 레이아웃은 375px 기준, 최대 너비 480px 카드 레이아웃
- **여백과 정돈**: 노션 다이어리 느낌 — 넓은 여백, 깔끔한 타이포, 모노톤 컬러
- **부담 없는 기록**: 모든 카테고리가 선택 사항 — 빈 카드가 있어도 괜찮음
- **빠른 접근**: URL 하나로 바로 현재 달 기록으로 진입
- **링크 프리뷰**: URL 입력 시 OG 태그 기반 카드 자동 생성으로 시각적 풍부함 제공

---

## 7. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | Next.js 14 (App Router) | React 기반, Vercel 배포 최적화, SSR 지원 |
| 스타일링 | Tailwind CSS | 빠른 모바일 퍼스트 UI 구성 |
| 백엔드/DB | Supabase (PostgreSQL) | 무료 티어로 개인 사용 충분, 내장 파일 스토리지 |
| 파일 스토리지 | Supabase Storage | 이미지 업로드 저장 |
| 배포 | Vercel | Next.js 최적화, 무료 티어 |
| OG 프리뷰 | 서버사이드 API 라우트 | 링크 입력 시 OG 메타 크롤링 |

## 8. 데이터 모델

### `entries` 테이블
```sql
id            uuid PRIMARY KEY DEFAULT gen_random_uuid()
year_month    char(7) NOT NULL         -- 예: '2026-04'
category      text NOT NULL            -- 예: 'song', 'photo', 'goal', ...
text_content  text                     -- 텍스트 내용
image_urls    text[]                   -- Supabase Storage URL 배열 (최대 3장)
link_url      text                     -- 외부 링크 URL
link_preview  jsonb                    -- { title, description, image, favicon }
memo          text                     -- 공통 메모 (최대 500자)
created_at    timestamptz DEFAULT now()
updated_at    timestamptz DEFAULT now()

UNIQUE (year_month, category)          -- 달별 카테고리 1개
```

### 카테고리 키 목록
`photo` / `song` / `goal` / `quote` / `restaurant` / `place` / `purchase` / `movie` / `discovery` / `gratitude`

## 9. 보안

- UUID 기반 URL: 환경 변수 `PRIVATE_UUID`로 관리, 소스 코드에 노출 안 됨
- Supabase Row Level Security (RLS): 모든 테이블에 RLS 활성화
  - 읽기/쓰기 모두 서버사이드 API 라우트를 통해서만 접근 (Supabase Service Role Key 사용)
  - 클라이언트에 Supabase 키 직접 노출 없음
- 이미지 업로드: 허용 파일 형식 검증 (jpg, png, webp, heic), 최대 10MB

## 10. 비기능 요구사항

- **성능**: 첫 화면 로딩 3초 이내 (모바일 LTE 기준)
- **이미지 최적화**: Next.js Image 컴포넌트 사용, WebP 변환
- **오프라인 대응**: 기본 에러 UI 제공 (오프라인 시 안내 메시지)
- **반응형**: 모바일(~480px) / 태블릿(~768px) / 데스크탑(~1280px) 모두 지원
- **다크 모드**: 시스템 설정 자동 감지 (`prefers-color-scheme`), Tailwind `dark:` 클래스 활용
- **PWA**: Next.js PWA 플러그인 적용 — 홈 화면 설치, 앱 아이콘, 스플래시 화면 지원
